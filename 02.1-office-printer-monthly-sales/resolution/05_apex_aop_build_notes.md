# Project 02.1 - APEX + AOP Build Notes

Use this guide to connect AOP and the delivered PL/SQL package with your existing app `POC_WEEKLY_SALES_EXCEL_UPLOAD`.

## 1) AOP Setup (First Time)

1. Install/configure the AOP plugin in your APEX environment.
2. Configure AOP server URL + API key at the correct level (instance/workspace/app, depending on your setup).
3. Validate with a basic "hello report" first.

Without this step, the report package can compile but PDF generation cannot run.

## 2) Delivered Database Artifacts

Run in order:

1. `01_reporting_objects.sql`
2. `02_pkg_monthly_sales_reports.pks`
3. `03_pkg_monthly_sales_reports.pkb`
4. `04_seed_and_test.sql`

## 3) Report Inputs (Recommended APEX Items)

Create report filter items on the report page:

- `P13_MONTH_START` (Date Picker, default `TRUNC(SYSDATE,'MM')`)
- `P13_REGION_CODE` (optional LOV from `POC_STORES.REGION_CODE`)
- `P13_TOP_N` (Number, default 10)
- `P13_RISK_THRESHOLD_PCT` (Number, default 20)

## 4) Report Data Sources

The package exposes one procedure per report:

- Report 1 (Summary by Rep): `pkg_monthly_sales_reports.report_summary_by_rep`
- Report 2 (Total Products Sold): `pkg_monthly_sales_reports.report_total_products`
- Report 3 (Top Stores + Risk Stores): `pkg_monthly_sales_reports.report_top_stores_risk`
- Shared KPI header: `pkg_monthly_sales_reports.report_kpi_header`

Use these as AOP data source queries/processes.

## 5) Standard Header/Footer Mapping

Use KPI/header dataset values and session values to fill:

- Company name
- Report name
- Period (`P13_MONTH_START`)
- Generated timestamp
- Generated by (`:APP_USER`)
- Page number

## 6) Authorization Check (Admin + Analyst)

You mentioned `ADMIN` and `ANALYST`. Confirm in APEX Shared Components:

- Authorization Schemes
- Scheme names
- Condition logic (role mapping / ACL table / custom function)

Apply authorization on:

- Monthly report page
- All print buttons/actions
- AJAX process that calls AOP

If your authorization is ACL-based, make both roles print-enabled. If only admin should print, apply admin scheme to print buttons and analyst scheme only to view regions.

## 7) Logging Hook (Recommended)

After each successful or failed report generation, call:

`pkg_monthly_sales_reports.log_report_run(...)`

This keeps a full execution trail for support and auditing.

## 8) Chart/Data Consistency Rule

For Report 2:

- Build chart from the same dataset returned by `report_total_products`.
- Do not use a separate query for chart values.

That guarantees chart and table totals match.
